# Рефакторинг сервисного слоя

Этот проект представляет собой пример рефакторинга сервисного слоя для подготовки к микросервисной архитектуре.

## Структура проекта

```
internal/
├── domain/                    # Доменная модель
│   ├── paste/                # Домен паст
│   │   ├── entity.go        # Сущности
│   │   ├── repository.go    # Интерфейсы репозитория
│   │   └── service.go       # Интерфейсы сервиса
│   └── auth/                # Домен аутентификации
│       ├── entity.go
│       ├── repository.go
│       └── service.go
│
├── service/                  # Реализации сервисов
│   ├── paste/               # Сервис паст
│   │   ├── service.go
│   │   └── dto.go
│   └── auth/                # Сервис аутентификации
│       ├── service.go
│       └── dto.go
│
└── infrastructure/          # Инфраструктурный слой
    ├── storage/            # Реализации хранилищ
    │   ├── minio/
    │   ├── redis/
    │   └── postgres/
    └── messaging/          # Очереди сообщений
```

## Основные принципы

1. **Разделение по доменам**:
   - Каждый домен независим
   - Домены общаются через события или API
   - Каждый домен имеет свою модель данных

2. **Инверсия зависимостей**:
   - Сервисы зависят от интерфейсов
   - Инфраструктурные детали изолированы

3. **Событийная архитектура**:
   - Асинхронное взаимодействие через события
   - Использование очередей сообщений

## План миграции

1. **Подготовка**:
   - Создать новую структуру проекта
   - Определить границы доменов
   - Спроектировать события

2. **Реализация доменов**:
   - Создать доменные модели
   - Определить интерфейсы репозиториев
   - Реализовать базовую логику

3. **Реализация сервисов**:
   - Перенести бизнес-логику
   - Добавить обработку событий
   - Реализовать DTO

4. **Инфраструктура**:
   - Реализовать репозитории
   - Настроить очереди сообщений
   - Добавить кэширование

5. **Тестирование**:
   - Написать unit-тесты
   - Провести интеграционное тестирование
   - Проверить производительность

## Преимущества нового подхода

1. **Масштабируемость**:
   - Независимое масштабирование компонентов
   - Горизонтальное масштабирование

2. **Поддерживаемость**:
   - Четкие границы между компонентами
   - Упрощенное тестирование
   - Легкость внесения изменений

3. **Гибкость**:
   - Возможность использования разных технологий
   - Простота добавления новых функций
   - Легкость замены компонентов
